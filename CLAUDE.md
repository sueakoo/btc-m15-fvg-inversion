# BTC M15 FVG Inversion Algorithm — Project Context

## Что это за проект

Веб-приложение (standalone HTML + JS, без бэкенда) для анализа торговой формации "инверсия FVG" на таймфрейме M15 фьючерсов BTCUSDT Perpetual (Binance).

**Цель:** пользователь находит инверсию на графике, вставляет данные свечей в приложение, получает детерминированный отчёт о качестве инверсии — без субъективности.

**НЕ является:** торговым советником, сигнальным ботом, авто-трейдером.

## Ключевые концепции

### Что такое инверсия FVG
- FVG (Fair Value Gap / Имбаланс) = трёхсвечная формация n-1, n, n+1
  - Медвежий FVG: low(n-1) > high(n+1). upper_fvg = low(n-1), lower_fvg = high(n+1)
  - Бычий FVG: high(n-1) < low(n+1). upper_fvg = low(n+1), lower_fvg = high(n-1)
- Инверсия лонг: свеча закрывается телом выше upper_fvg медвежьего FVG (close > upper_fvg)
- Инверсия шорт: свеча закрывается телом ниже lower_fvg бычьего FVG (close < lower_fvg)
- Если close вышел за границу хотя бы на 1 пункт = факт инверсии
- Хвост за границей НЕ считается инверсией

### Ключевые окна анализа
- **Pivot** = разворотная свеча (экстремум перед движением к FVG)
- **OI_window** = свечи после Pivot → свеча инверсии включительно (Pivot НЕ входит)
- **FVG_overlap_window** = только свечи фактического перекрытия FVG
- **H1_window** = H1 свечи, покрывающие тот же период (следующая M15 после Pivot → инверсия)

### H1 данные
- H1 используется для ликвидаций и верификации (M15 ликвидации ненадёжны, часто #N/A)
- Берутся все H1 свечи, чей период пересекается с M15 окном
- Незакрытая H1 берётся как snapshot (данные на текущий момент)
- Количество H1 свечей: переменное, обычно 1-4

### Edge case: single_candle_inversion
- Если FVG перекрыта одной свечой (она же свеча инверсии): Re-Auction анализирует эту свечу
- Флаг `single_candle_inversion = true` выводится в отчёте

## Формат входных данных

Пользователь вставляет текстовые блоки вида:
```
ts: 15.06.2026 17:15
exchange: Binance
symbol: BTCUSDT Perp
tf: 15m
open: 75635.60
high: 75680.10
...
doi_pct: 0.14%
implied_price: 75653.00
avg_trade_buy: 0.03
avg_trade_sell: 0.06
```

Несколько свечей подряд. M15 и H1 вставляются раздельно в разные поля.
Парсер разбивает по `ts:`, определяет TF по полю `tf:`.

### Единицы данных
- `buy_volume` / `sell_volume` = BTC (базовый актив)
- `volume` = USDT (quote volume)
- `oi_open` / `oi_close` = BTC/контракты (не USD)
- `liq_long` / `liq_short` = USDT (ликвидации)
- `doi_pct` = изменение OI в % (ΔOI / OI_prev × 100)
- `implied_price` = OI-взвешенная имплицитная цена (из X-RAY, не вычисляем)
- `cvd_pct` = кумулятивная дельта в % от объёма
- `liqshare_pct` = доля ликвидаций от объёма (в %)
- `limb_pct` = дисбаланс ликвидаций (какая сторона преобладает)
- `oe` = OE = doi_pct / ABS(Δ% цены) — вспомогательная, из X-RAY, только в комментарий

### Вердикты по итоговому баллу:
- 76–100: Сильный сетап
- 60–75: Рабочий сетап
- 45–59: Стандартная инверсия
- 30–44: Ослабленная инверсия
- < 30 или стоп-флаг: Не брать

### Стоп-флаги (7 штук) — любой = "НЕ БРАТЬ":
1. Пустая инверсия (нет OI + нет объёма + H1 не подтверждает)
2. Слабый OI без передачи риска (retention < 0.15, нет объёмной структуры)
3. Пустое FVG + слабый общий ресурс
4. Аномальный Skew: skew_depth > 1.20 ИЛИ implied_price < pivot_low (для лонга)
5. H1 вынос стопов без живого интереса (liqshare >= 10% + OI падает)
6. Все основные блоки одновременно провалены
7. **Системная пустота** (новый): Energy ≤ 4 И Retention ≤ 4 И Re-Auction ≤ 3 И H1 ≤ 3

### Правила минимального порога:
- Если Energy ≤ 4 И Retention ≤ 4 → красный флаг "Пустой OI-ресурс" (не стоп, но в отчёт)
- Если выше + Re-Auction ≤ 3 + H1 ≤ 3 → Стоп-флаг #7

## Ключевые метрики

```
gross_oi = сумма max(doi_pct, 0) внутри OI_window
unload_oi = сумма min(doi_pct, 0) внутри OI_window
net_oi = gross_oi + unload_oi
retention_ratio = net_oi / gross_oi

# Блок 2 — ценовые зоны OI Placement:
share_below = Σ(max(doi_pct,0) для свечей где high < lower_fvg) / gross_oi
share_fvg   = Σ(max(doi_pct,0) для свечей перекрывающих [lower_fvg, upper_fvg]) / gross_oi
share_inv   = max(doi_pct[inversion_candle], 0) / gross_oi

fvg_volume_share = volume_FVG_overlap / post_pivot_volume
final_volume_share = volume_inversion_candle / post_pivot_volume

imb_range_pct = (upper_fvg - lower_fvg) / mid_fvg × 100

skew_depth = (close_inv - implied_price) / (upper_fvg - lower_fvg)  [лонг]
           = (implied_price - close_inv) / (upper_fvg - lower_fvg)  [шорт]

# H1 агрегация:
h1_doi_pct = Σ(doi_pct) по всем H1 свечам окна
h1_liqshare_pct = (Σliq_long + Σ|liq_short|) / Σvolume × 100
h1_limb_pct = (Σliq_long − Σ|liq_short|) / (Σliq_long + Σ|liq_short|) × 100
h1_cvd_sign = знак(Σ cvd_pct)
```

## Логика блоков — утверждённые решения

### Блок 0 — Detection

**Pivot (авто-определение):**
- Лонг: min LOW среди свечей где low < lower_fvg (ниже зоны FVG), до инверсии. При равных → берём более позднюю.
- Шорт: max HIGH среди свечей где high > upper_fvg (выше зоны FVG), до инверсии. При равных → берём более позднюю.

**Несколько FVG в данных:**
- Алгоритм находит первую инверсию → анализирует только её, остальное игнорирует.
- Если два FVG одного направления в одном движении (без разворота между ними) → объединяем в одну зону: upper_fvg = max(upper_fvg), lower_fvg = min(lower_fvg). Один отчёт.

**FVG размеры (imb_range_pct):**
- < 0.15% = малый FVG
- 0.15–0.40% = средний FVG
- > 0.40% = большой FVG

### Блок 2 — OI Placement (11 pts)

**Отвечает на вопрос:** где сосредоточен новый OI относительно FVG — у кого есть подушка перед тестом, кто сидит на максимуме.

**Логика:** ценовые зоны (не счётчик свечей):
- **Zone_below**: свечи где high < lower_fvg (цена не касалась FVG)
- **Zone_fvg**: свечи, чей диапазон перекрывает [lower_fvg, upper_fvg]
- **Zone_inv**: свеча инверсии (close > upper_fvg для лонга)

```
share_below = Σ(max(doi_pct,0) в Zone_below) / gross_oi
share_fvg   = Σ(max(doi_pct,0) в Zone_fvg)   / gross_oi
share_inv   = max(doi_pct[inv], 0)             / gross_oi
```

**Таблица баллов:**
| share_below + share_fvg | Уточнение | Балл |
|------------------------|-----------|------|
| ≥ 0.80 | share_fvg ≥ 0.40 | 11 |
| ≥ 0.80 | share_fvg < 0.40 | 10 |
| 0.60–0.79 | share_fvg ≥ 0.30 | 9 |
| 0.60–0.79 | share_fvg < 0.30 | 8 |
| 0.40–0.59 | share_fvg ≥ 0.20 | 6 |
| 0.40–0.59 | share_fvg < 0.20 | 5 |
| 0.20–0.39 | share_fvg ≥ 0.10 | 4 |
| 0.20–0.39 | share_fvg < 0.10 | 3 |
| < 0.20 | share_fvg ≥ 0.05 | 2 |
| < 0.20 | share_fvg < 0.05 | 1 |
| gross_oi < 0.20% | cap max 4 | ≤ 4 |
| OI_window ≤ 3 свечей | cap max 8 | ≤ 8 |
| gross_oi = 0 | — | 0 |

### Блок 3 — OI Retention (13 pts)

**Отвечает на вопрос:** сколько живого OI осталось после завершения инверсии — есть ли кому защищать зону при тесте.

**Входные данные:** net_oi, retention_ratio, fvg_volume_share.
**НЕ использует:** gross_oi как первичный (только контекст в передаче риска).
**НЕ использует:** паттерны порядка событий (нельзя определить сторону OI-снятий без структурного слома).

**Передача риска** (отдельный сценарий):
Условие: net_oi < 0.20% + gross_oi ≥ 0.40% + fvg_volume_share ≥ 35%
→ Высокая активность в зоне при слабом чистом OI. Вероятная смена позиции. Прямого подтверждения нет — отмечается в комментарии.

**2D матрица баллов (net_oi × retention_ratio):**
| | retention ≥ 0.70 | retention 0.40–0.70 | retention 0.15–0.40 | retention < 0.15 |
|---|---|---|---|---|
| **net_oi ≥ 0.40%** | 13 | 11 | 9 | 7 |
| **net_oi 0.20–0.39%** | 11 | 9 | 7 | 5 |
| **net_oi 0.10–0.19%** | 7 | 5 | 3 | 2 |
| **net_oi < 0.10%** | 4 | 3 | 2 | 1 |
| **gross_oi = 0** | 0 | 0 | 0 | 0 |

**Передача риска (особый сценарий):** 9 pts (независимо от матрицы)

### Блок 4 — Механика захвата (16 pts)

*(Объединение бывших блоков 4 и 7. Блок 7 как отдельный не существует.)*

**Отвечает на вопрос:** каким механизмом цена прошла через FVG и инвертировала зону.

**Вспомогательные метрики (только в комментарий, не в скоринг):** OE, avg_trade_buy/sell, ER.

**Классификация тела свечи (связка body_pct + CVD):**
- body_pct ≥ 50% + clv в сторону инверсии ≥ 60% = сильное направленное тело
- body_pct < 35% + сильный CVD в сторону инверсии = "Инверсия через сопротивление"
- body_pct < 35% + слабый CVD = "Движение без позиции"

**Сценарии:**
| Сценарий | Что произошло | Балл |
|---------|---------------|------|
| Лимитное поглощение | CVD против цены или нейтральный, цена выросла, OI набирается. Крупный лимит поглощал продажи. | 13–16 |
| Рыночная инициатива | CVD и цена в одну сторону, OI растёт. Прямое направленное давление. | 11–13 |
| Инверсия через сопротивление | CVD сильный в сторону инверсии, тело слабое — встречная стена была, но цена прошла. OI набирается. | 9–11 |
| Движение без позиции | Цена прошла FVG, CVD слабый, OI не набирается. Движение без позиции. | 4–7 |
| Пустое движение | Вакуум с другой стороны. CVD нет, OI нет. | 0–3 |
| Возможный хедж | OI растёт, цена почти не двигается, CVD слабый/смешанный. | 0–2 |

### Блок 5 — Re-Auction (13 pts)

**Отвечает на вопрос:** как FVG зона была принята в процессе прохождения — был ли реальный аукцион внутри зоны.

**Особые случаи:**

**single_candle_inversion (одна свеча закрыла весь FVG):**
- fvg_volume_share = null (не считается — 100% по определению)
- Анализируем: doi_pct этой свечи > 0? CVD в нужную сторону?
- Максимальный балл: **7/13**
- Вывод: "Инверсия однослайновая — объёмное распределение нерелевантно"

**candle_count_fvg = 1:** max 7/13, CVD бинарный (подтверждает / не подтверждает)
**candle_count_fvg = 2:** max 9/13, CVD бинарный (убираем "mixed")
**candle_count_fvg ≥ 3:** стандартная логика, CVD может быть mixed

**fvg_volume_share порог:**
- Стандартно (≥ 3 свечей в FVG): ≥ 50% = выраженное участие
- При candle_count_fvg ≤ 2: порог поднят до **≥ 70%** = выраженное участие

### Блок 6 — Geometry (8 pts)

**Отвечает на вопрос:** насколько чисто выглядит геометрия движения — риск-сигнал, не показатель качества.

- Анализ начинается со свечи ПОСЛЕ Pivot (Pivot исключён)
- directional_tail_risk_avg > 50% → **минимум 2 балла** (не 0), + флаг "Кульминационный хвост: повышен риск глубокого теста"
- Слабая геометрия при сильном OI/flow → флаг, не инвалидация

### Блок 8 — Skew (7 pts)

**Отвечает на вопрос:** куда тянет алгоритмическое позиционирование — поддерживает ли implied_price инверсию или работает против.

**Формула:**
```
skew_depth = (close_inv - implied_price) / (upper_fvg - lower_fvg)  [лонг]
skew_depth = (implied_price - close_inv) / (upper_fvg - lower_fvg)  [шорт]
```

implied_price берётся из X-RAY как готовое поле, не вычисляется.

**Шкала:**
| skew_depth | Балл |
|------------|------|
| ≤ 0 | 7 |
| 0–0.15 | 6 |
| 0.15–0.33 | 5 |
| 0.33–0.50 | 4 |
| 0.50–0.75 | 3 |
| 0.75–1.00 | 2 |
| 1.00–1.20 | 1 |
| > 1.20 | 0 + СТОП-ФЛАГ |

**Дополнительный стоп-флаг:** implied_price < pivot_low (для лонга) → стоп независимо от skew_depth.

**Skew trend:** только флаги/комментарий, НЕ влияет на балл.

### Блок 9 — H1 Snapshot (18 pts)

**Отвечает на вопрос:** что происходило на H1 в тот же период — верификация через старший таймфрейм.

**Агрегация H1 данных:**
```python
h1_doi_pct    = sum(doi_pct для всех H1 свечей окна)
h1_liqshare   = (sum(liq_long) + sum(abs(liq_short))) / sum(volume) * 100
h1_limb_pct   = (sum(liq_long) - sum(abs(liq_short))) / (sum(liq_long) + sum(abs(liq_short))) * 100
h1_cvd_sign   = sign(sum(cvd_pct))
```

**Матрица сценариев:**
| h1_liqshare | H1 OI (h1_doi_pct) | limb | Сценарий | Балл |
|------------|-------------------|------|----------|------|
| < 2% | ≥ 0.20% | любое | Чистый рост: новый OI без стопов | 18 |
| < 2% | 0–0.19% | любое | Слабый рост: OI почти не появился | 13 |
| < 2% | < 0 | любое | OI уходит, H1 не поддерживает | 5 |
| 2–5% | ≥ 0.20% | против инверсии | Чужие стопы помогли + OI вошёл | 14 |
| 2–5% | ≥ 0.20% | в сторону инверсии | Встряска + OI удержался | 11 |
| 2–5% | < 0.20% | любое | H1 нейтрален, подтверждения нет | 7 |
| 5–10% | ≥ 0.20% | против инверсии | Выбитые стопы + новый OI | 13 |
| 5–10% | ≥ 0.20% | в сторону инверсии | Паника в зоне + OI удержался | 11 |
| 5–10% | 0–0.19% | любое | Ликвидации без роста OI | 7 |
| 5–10% | < 0 | любое | Ликвидации + OI снижается | 4 |
| ≥ 10% | ≥ 0.20% | любое | Сильные ликвидации + OI набирается | 14 |
| ≥ 10% | 0–0.19% | любое | Высокая доля стопов, OI не растёт | 2 |
